Painel Administrativo

Cockpit da loja: pedidos, clientes, cliques, produtos, pagamento, frete, cupons, contato, API e documentação.

Cockpit de gestão da loja Sensor Tattoo Fix — por prioridade:

  • Pedidos — lista, etiquetas, PIX e exportação na aba Pedidos.
  • Clientes — contas criadas em Minha Conta.
  • Cliques — log de visitas e links (ML, TikTok, loja…).
  • Produtos — lentes na vitrine e agregados no checkout (preço, peso, imagem).
  • Pagamento — cartão, PIX e PayPal internacional (processadores e chave PIX reserva).
  • Frete — remetente, modalidades, motoboy, Correios BR e internacional.
  • Cupons — desconto ao cliente e comissão ao comissionado.
  • Contato — WhatsApp, e-mails da loja, assuntos, mensagens e testes de envio.
  • API — URL do Worker e status das integrações (MP, Asaas, PayPal, Correios…).
  • Documentação — guia técnico (secrets, webhooks, deploy).

Login: usuário admin + senha do secret ADMIN_PASSWORD no Cloudflare. Após 5 erros, bloqueio ~30 min.

Modalidades de envio (cliente escolhe)

O checkout lista preço e prazo de cada modalidade ativa.

Internacional: tipo M = Exporta Fácil (encomenda); tipo D = documento/carta internacional (mais barato para lente fina). Use * para todos os serviços daquele tipo.

Envio particular (motoboy)

Cotação por distância (CEP origem → destino): taxa fixa + valor por km (arredondado para cima).

Ao confirmar o pagamento, cada motoboy ativo recebe e-mail com o pedido. Marque Módulo motoboy ativo e cadastre ao menos um entregador — a modalidade no checkout é ligada automaticamente ao salvar.

                         Frete internacional
                        

                            
O checkout usa o simulador Exporta Fácil em tempo real. A tabela abaixo é fallback (se a API cair) e é atualizada automaticamente quando a API responde.

                            
Acréscimo internacional: somado só na primeira opção de documento/carta (a mais barata).                          Cartão de crédito (Brasil)
                        

                            
Define qual gateway processa cartão de crédito no checkout nacional (Asaas ou Mercado Pago).

                            
Secrets no Worker: ASAAS_API_KEY e/ou MP_ACCESS_TOKEN.

                            
Webhooks: /webhook/asaas e /webhook/mercadopago

                        

                        

                            
                            
                        

                    


                    

                         PIX no checkout
                        

                            
Processador de PIX independente do cartão. Secrets no Worker: MP_ACCESS_TOKEN e/ou ASAAS_API_KEY.

                            
Se o processador principal falhar e o fallback estiver ativo, tenta o outro (Asaas ↔ Mercado Pago).

                            
Se ambos falharem, o checkout usa o PIX reserva abaixo (QR estático + botão copiar).

                            
Conexão real com Mercado Pago/Asaas: aba API → Status das integrações.

                        

                        

                            
                            
                        

                    


                    

                         PIX reserva (QR estático)
                        

                            
Usado somente se Mercado Pago e Asaas falharem. O QR aponta para a chave abaixo; nome e cidade aparecem no comprovante do banco.

                            
Se a chave estiver inválida ou vazia, o site usa o cadastro em data/store-config.json (CNPJ da empresa).

                            
Chave: CNPJ, e-mail ou telefone cadastrados no banco — não confundir com e-mail de notificação de pedidos.

                        

                        

                        

                            
                            
                            
                            
                        

                    


                    

                         PayPal
                        

                            
PayPal no checkout: Brasil (envio nacional) e internacional (país ≠ Brasil).

                            
Nome da conta: rótulo só no admin para identificar qual PayPal está conectado.

                            
Secrets no Worker: PAYPAL_CLIENT_ID, PAYPAL_CLIENT_SECRET, PAYPAL_SANDBOX=false

                            
Webhook: /webhook/paypal — confirmação automática após pagamento.
